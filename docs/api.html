API Reference

OpenHoof exposes a local REST API and WebSocket endpoint for programmatic access. Use it to send messages, manage memory, control skills, and integrate with your own tools.

Base URL

All API endpoints are served from the daemon's HTTP server:

http://localhost:3777/api/v1

The port is configurable via daemon.port in your config. All responses are JSON unless otherwise noted.

Authentication

By default, the API has no authentication (it only binds to localhost). If you expose the daemon to your network, enable API key authentication:

{
  "daemon": {
    "host": "0.0.0.0",
    "apiKey": "your-secret-api-key"
  }
}

Include the key in requests via the Authorization header:

curl -H "Authorization: Bearer your-secret-api-key" \
  http://localhost:3777/api/v1/status

Chat

POST /api/v1/chat

Send a message and receive an AI response.

curl -X POST http://localhost:3777/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What is the capital of France?",
    "conversationId": "conv_abc123",
    "agent": "default"
  }'

Request body:

FieldTypeRequiredDescription
messagestringyesThe user's message
conversationIdstringnoContinue an existing conversation. Omit to start a new one
agentstringnoNamed agent to use. Defaults to the default agent
streambooleannoEnable SSE streaming (default: false)

Response:

{
  "conversationId": "conv_abc123",
  "message": {
    "role": "assistant",
    "content": "The capital of France is Paris.",
    "model": "claude-sonnet-4-20250514",
    "tokens": { "input": 24, "output": 12 }
  }
}

POST /api/v1/chat (Streaming)

Set "stream": true to receive Server-Sent Events:

curl -N -X POST http://localhost:3777/api/v1/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "Write a haiku", "stream": true}'
data: {"type":"start","conversationId":"conv_xyz"}
data: {"type":"delta","content":"Silent"}
data: {"type":"delta","content":" morning"}
data: {"type":"delta","content":" dew"}
...
data: {"type":"done","tokens":{"input":12,"output":28}}

WebSocket

For real-time bidirectional communication, connect via WebSocket:

const ws = new WebSocket('ws://localhost:3777/ws');

ws.onopen = () => {
  ws.send(JSON.stringify({
    type: 'chat',
    message: 'Hello from WebSocket!',
    conversationId: 'conv_ws_001'
  }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  if (data.type === 'delta') {
    process.stdout.write(data.content);
  } else if (data.type === 'done') {
    console.log('\n[Done]');
  }
};

Memory

GET /api/v1/memory/search?q={query}

Search memory with semantic similarity:

curl "http://localhost:3777/api/v1/memory/search?q=user+preferences&limit=5"
{
  "results": [
    {
      "id": "mem_001",
      "content": "User prefers dark mode in all applications",
      "similarity": 0.89,
      "createdAt": "2025-06-15T10:30:00Z",
      "namespace": "default"
    }
  ]
}

POST /api/v1/memory

Add a memory entry:

curl -X POST http://localhost:3777/api/v1/memory \
  -H "Content-Type: application/json" \
  -d '{
    "content": "Project deadline is March 30th",
    "namespace": "work"
  }'

DELETE /api/v1/memory/:id

Delete a memory entry by ID:

curl -X DELETE http://localhost:3777/api/v1/memory/mem_001

Skills

GET /api/v1/skills

List all installed skills:

curl http://localhost:3777/api/v1/skills
{
  "skills": [
    {
      "name": "web-browse",
      "version": "1.2.0",
      "enabled": true,
      "tools": ["browse_url", "extract_text", "screenshot_page"]
    },
    {
      "name": "code-exec",
      "version": "1.0.0",
      "enabled": true,
      "tools": ["execute_code", "run_shell"]
    }
  ]
}

POST /api/v1/skills/:name/execute

Execute a skill tool directly:

curl -X POST http://localhost:3777/api/v1/skills/web-browse/execute \
  -H "Content-Type: application/json" \
  -d '{
    "tool": "browse_url",
    "params": {
      "url": "https://example.com",
      "extractText": true
    }
  }'

System

GET /api/v1/status

Get daemon status and system information:

curl http://localhost:3777/api/v1/status
{
  "status": "running",
  "version": "1.4.2",
  "uptime": 86400,
  "agent": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-20250514"
  },
  "channels": {
    "webchat": { "status": "connected" },
    "telegram": { "status": "connected" },
    "discord": { "status": "disconnected" }
  },
  "memory": {
    "entries": 1247,
    "backend": "local"
  }
}

POST /api/v1/config/reload

Hot-reload the configuration without restarting the daemon:

curl -X POST http://localhost:3777/api/v1/config/reload

Scheduler

GET /api/v1/schedules

List all scheduled tasks:

curl http://localhost:3777/api/v1/schedules

POST /api/v1/schedules

Create a new scheduled task:

curl -X POST http://localhost:3777/api/v1/schedules \
  -H "Content-Type: application/json" \
  -d '{
    "cron": "0 9 * * 1-5",
    "task": "Summarize my unread emails and create a to-do list",
    "agent": "default",
    "channel": "telegram"
  }'

Error Responses

All errors follow a consistent format:

{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The 'message' field is required",
    "status": 400
  }
}

Common error codes:

CodeStatusDescription
INVALID_REQUEST400Missing or invalid request parameters
UNAUTHORIZED401Missing or invalid API key
NOT_FOUND404Resource not found
AGENT_ERROR502The upstream AI provider returned an error
RATE_LIMITED429Too many requests (provider rate limit)
