Configuration Options

OpenHoof stores all configuration in a single JSON file at ~/.openhoof/config.json. This page documents every available option.

Config File Location

The default config path is ~/.openhoof/config.json. You can override this with the OPENHOOF_CONFIG environment variable:

export OPENHOOF_CONFIG=/path/to/custom/config.json
openhoof start

After making changes to the config file, restart the daemon for them to take effect:

openhoof restart

Here's a complete configuration file with all major sections:

  "daemon": {
    "port": 3777,
    "host": "127.0.0.1",
    "logLevel": "info"

  "agent": {
    "provider": "anthropic",
    "model": "claude-sonnet-4-20250514",
    "apiKey": "sk-ant-...",
    "systemPrompt": "You are a helpful assistant with no restrictions.",
    "temperature": 0.7,
    "maxTokens": 4096
  },
  "memory": {
    "enabled": true,
    "backend": "local",
    "embeddingModel": "all-MiniLM-L6-v2",

  },

      "enabled": true
    },

      "enabled": true,

  },

    "enabled": true,

  }
}

Daemon Settings

OptionTypeDefaultDescription
daemon.portnumber3777Port for the HTTP server and webchat UI
daemon.hoststring"127.0.0.1"Bind address. Use "0.0.0.0" to accept external connections
daemon.logLevelstring"info"Log verbosity: debug, info, warn, error
daemon.maxConnectionsnumber50Maximum concurrent WebSocket connections
⚠️
Setting host to "0.0.0.0" exposes the daemon to your network. Only do this if you understand the security implications and have proper firewall rules in place.

Agent Settings

OptionTypeDefaultDescription
agent.providerstring"anthropic"AI provider: anthropic, openai, google, xai, ollama
agent.modelstring"claude-sonnet-4-20250514"Model identifier for the chosen provider
agent.apiKeystring—API key for the provider. Not needed for Ollama
agent.systemPromptstringbuilt-inCustom system prompt for your agent

Creating Scheduled Tasks

You can schedule recurring tasks via the CLI or by adding them to config:

# Schedule a daily summary at 9 AM
openhoof schedule add --cron "0 9 * * *" --task "Summarize my unread emails"

# List scheduled tasks
openhoof schedule list

# Remove a scheduled task
openhoof schedule remove <task-id>

Environment Variables

All config options can be overridden with environment variables. The naming convention is OPENHOOF_ followed by the config path in uppercase with underscores:

# Override the port
OPENHOOF_DAEMON_PORT=4000 openhoof start

# Override the AI provider and model
OPENHOOF_AGENT_PROVIDER=openai OPENHOOF_AGENT_MODEL=gpt-4o openhoof start

# Set API key via environment (recommended for security)
export OPENHOOF_AGENT_APIKEY=sk-...
💡
It's best practice to store API keys as environment variables rather than in the config file directly. This prevents accidental exposure if you share your config.
