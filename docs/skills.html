Skills & Plugins

Skills are modular plugins that extend your agent's capabilities. They give your AI new tools — from web browsing and code execution to image generation and file management.

How Skills Work

Skills are tool definitions that get injected into your agent's context. When the AI determines it needs to use a skill (like browsing a webpage or running code), it calls the skill function, and OpenHoof executes it locally on your machine. The result is passed back to the AI for its response.

Each skill is a self-contained module with a manifest that describes its capabilities, required permissions, and configuration options. Skills run in their own process sandbox by default, though you can disable sandboxing for trusted skills.

Built-in Skills

OpenHoof ships with several skills pre-installed:

  • web-browse — Browse any URL and extract content. Uses a headless Chromium instance for full JavaScript rendering.
  • code-exec — Execute code in Python, Node.js, Bash, or any installed language runtime. Runs in a subprocess with configurable timeouts.
  • file-manager — Read, write, move, and delete files on your filesystem. The agent can manage your files directly.
  • web-search — Search the web via DuckDuckGo, Google, or Brave Search APIs.
  • shell — Execute arbitrary shell commands. Full terminal access for your AI.
  • screenshot — Capture screenshots of your desktop or specific windows.

Installing Skills

Install community skills from the OpenHoof skill registry:

# Install a skill by name
openhoof skill install image-gen

# Install from a git repository
openhoof skill install https://github.com/user/openhoof-skill-example.git

# Install from a local directory
openhoof skill install ./my-custom-skill

Skills are installed to ~/.openhoof/skills/ by default.

Managing Skills

# List installed skills
openhoof skill list

# Get info about a skill
openhoof skill info web-browse

# Update all skills
openhoof skill update

# Remove a skill
openhoof skill remove image-gen

# Disable a skill without removing it
openhoof skill disable image-gen

# Re-enable a disabled skill
openhoof skill enable image-gen

Writing Custom Skills

Create your own skills to give your agent custom capabilities. A skill is a directory with a manifest and an entry point:

# Scaffold a new skill
openhoof skill create my-skill

This creates the following structure:

my-skill/
  manifest.json    # Skill metadata and tool definitions
  index.js         # Entry point
  package.json     # Dependencies (optional)

Manifest File

The manifest.json describes your skill and its tools:

{
  "name": "my-skill",
  "version": "1.0.0",
  "description": "A custom skill that does something useful",
  "author": "your-name",
  "tools": [
    {
      "name": "do_something",
      "description": "Performs a specific action with the given input",
      "parameters": {
        "type": "object",
        "properties": {
          "input": {
            "type": "string",
            "description": "The input to process"
          },
          "format": {
            "type": "string",
            "enum": ["json", "text", "markdown"],
            "description": "Output format"
          }
        },
        "required": ["input"]
      }
    }
  ],
  "permissions": ["filesystem", "network"],
  "sandbox": true
}

Entry Point

The index.js file exports handler functions matching your tool names:

module.exports = {
  async do_something({ input, format = "text" }) {
    // Your skill logic here
    const result = await processInput(input);

    if (format === "json") {
      return JSON.stringify(result, null, 2);
    }

    return result.toString();
  }
};

async function processInput(input) {
  // Process the input and return a result
  return { processed: true, data: input.toUpperCase() };
}

Skill Permissions

Skills declare the permissions they need in the manifest:

PermissionDescription
filesystemRead and write files
networkMake HTTP requests and open sockets
shellExecute shell commands
browserControl the headless browser
clipboardAccess the system clipboard
notificationsSend desktop notifications

Skill Configuration

Skills can have custom config options in your main config file:

{
  "skills": {
    "autoLoad": true,
    "directory": "~/.openhoof/skills",
    "config": {
      "web-browse": {
        "headless": true,
        "timeout": 30000,
        "userAgent": "OpenHoof/1.0"
      },
      "code-exec": {
        "timeout": 60000,
        "maxOutputSize": "10MB"
      }
    }
  }
}
