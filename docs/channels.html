Connecting Channels

Channels let you talk to your AI agent through the messaging apps you already use. OpenHoof supports Telegram, Discord, WhatsApp, Slack, Signal, iMessage, and LINE out of the box.

How Channels Work

Each channel is a bidirectional bridge between a messaging platform and your OpenHoof daemon. When a message arrives through any channel, it's routed to your configured AI agent. The response is streamed back through the same channel.

You can run multiple channels simultaneously. A message from Telegram and a message from Discord both reach the same agent, with the same memory and skills. Channel-specific context (like the sender's username) is passed to the agent automatically.

Webchat (Built-in)

The webchat UI is enabled by default at http://localhost:3777. No configuration needed — it works as soon as the daemon starts. The webchat supports markdown rendering, code highlighting, file uploads, and streaming responses.

{
  "channels": {
    "webchat": {
      "enabled": true,
      "title": "OpenHoof Chat",
      "theme": "dark"
    }
  }
}

Telegram

Connect your AI to a Telegram bot. Users can message it directly or add it to group chats.

Setup

  1. Message @BotFather on Telegram and create a new bot with /newbot
  2. Copy the bot token
  3. Add it to your config:
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "123456789:ABCdefGhIjKlMnOpQrStUvWxYz",
      "allowedUsers": [],
      "allowGroups": true
    }
  }
}

Set allowedUsers to an array of Telegram user IDs to restrict access. Leave empty to allow anyone. Set allowGroups to false to disable group chat responses.

# Or configure via CLI
openhoof channel add telegram --token "123456789:ABCdef..."

Discord

Run your AI as a Discord bot in your server.

Setup

  1. Go to the Discord Developer Portal and create an application
  2. Under "Bot", create a bot and copy the token
  3. Under "OAuth2 > URL Generator", select bot scope with "Send Messages" and "Read Message History" permissions
  4. Use the generated URL to invite the bot to your server
{
  "channels": {
    "discord": {
      "enabled": true,
      "botToken": "MTIz...",
      "allowedChannels": ["general", "ai-chat"],
      "respondToMentions": true
    }
  }
}

When respondToMentions is true, the bot only responds in group channels when @mentioned. It always responds in DMs.

WhatsApp

Connect via the WhatsApp Web protocol. This uses a local browser session — no Business API needed.

{
  "channels": {
    "whatsapp": {
      "enabled": true,
      "sessionPath": "~/.openhoof/whatsapp-session",
      "allowedNumbers": ["+1234567890"]
    }
  }
}
# Start the WhatsApp connection (shows QR code to scan)
openhoof channel connect whatsapp

On first connect, a QR code is displayed in the terminal. Scan it with WhatsApp on your phone. The session persists until you disconnect.

Slack

Add your AI as a Slack app in your workspace.

{
  "channels": {
    "slack": {
      "enabled": true,
      "botToken": "xoxb-...",
      "appToken": "xapp-...",
      "allowedChannels": ["ai-assistant"]
    }
  }
}

You'll need to create a Slack app with Socket Mode enabled. Required scopes: chat:write, app_mentions:read, im:history, im:read, im:write.

Signal

Connect to Signal via the signal-cli bridge:

{
  "channels": {
    "signal": {
      "enabled": true,
      "phoneNumber": "+1234567890",
      "signalCliPath": "/usr/local/bin/signal-cli"
    }
  }
}
ℹ️
Signal requires signal-cli to be installed separately. Install it via your package manager: brew install signal-cli (macOS) or apt install signal-cli (Linux).

iMessage

macOS only. Uses AppleScript to send and receive iMessages:

{
  "channels": {
    "imessage": {
      "enabled": true,
      "pollInterval": 2000,
      "allowedContacts": []
    }
  }
}

Requires macOS with Messages app configured. Grant OpenHoof accessibility permissions when prompted.

LINE

Connect to LINE via the Messaging API:

{
  "channels": {
    "line": {
      "enabled": true,
      "channelAccessToken": "...",
      "channelSecret": "..."
    }
  }
}

Create a LINE Messaging API channel at the LINE Developers Console. Set the webhook URL to your daemon's public endpoint.

Channel CLI Commands

# List all configured channels
openhoof channel list

# Add a new channel interactively
openhoof channel add <type>

# Test a channel connection
openhoof channel test <type>

# Remove a channel
openhoof channel remove <type>
