Installation Guide

Install OpenHoof globally via npm and get the daemon running on your machine in minutes. Supports macOS, Linux, and Windows via WSL.

Prerequisites

Before installing OpenHoof, make sure you have the following:

  • Node.js v18+ — We recommend using the latest LTS release. Check your version with node -v.
  • npm v9+ — Ships with Node.js. Verify with npm -v.
  • An AI provider API key — You'll need at least one: Anthropic (Claude), OpenAI (GPT), Google (Gemini), xAI (Grok), or a local Ollama instance.
ℹ️
On Windows, OpenHoof must run inside WSL (Windows Subsystem for Linux). Native Windows support is not available due to process management dependencies.

Install via npm

The recommended installation method is via npm global install:

# Install OpenHoof globally
npm i -g openhoof

# Verify the installation
openhoof --version

This installs the openhoof CLI binary and all dependencies. The binary is added to your system PATH automatically.

Alternative: Install from source

If you prefer to run from source or want to contribute:

# Clone the repository
git clone https://github.com/openclaw/openhoof.git
cd openhoof

# Install dependencies
npm install

# Build the project
npm run build

# Link the CLI globally
npm link

Initialize Configuration

After installation, run the init command to create your configuration file:

openhoof init

This creates ~/.openhoof/config.json with sensible defaults and walks you through an interactive setup:

  1. Choose a default AI provider — Select from Claude, GPT, Gemini, Grok, or Ollama
  2. Enter your API key — Stored locally in your config file, never transmitted elsewhere
  3. Select a default model — Pick the specific model variant (e.g., claude-sonnet-4-20250514)
  4. Set the daemon port — Defaults to 3777

You can always edit the config manually later. See Configuration for all available options.

Start the Daemon

Launch the OpenHoof daemon:

# Start in the background
openhoof start

# Or run in the foreground (useful for debugging)
openhoof start --foreground

The daemon starts listening on http://localhost:3777. Open this URL in your browser to access the built-in webchat UI.

Daemon Management

# Check if the daemon is running
openhoof status

# View real-time logs
openhoof logs -f

# Restart the daemon
openhoof restart

# Stop the daemon
openhoof stop

Auto-Start on Boot

To have OpenHoof start automatically when your machine boots:

macOS (launchd)

openhoof service install
# This creates a LaunchAgent plist and loads it

Linux (systemd)

openhoof service install
# This creates and enables a systemd user service

To remove the auto-start service:

openhoof service uninstall

Updating

Update to the latest version with npm:

# Update OpenHoof
npm update -g openhoof

# Restart the daemon to use the new version
openhoof restart
⚠️
After updating, always restart the daemon. The running daemon uses the old binary until restarted. Check the changelog for breaking changes before major version upgrades.

Uninstalling

To completely remove OpenHoof:

# Remove the auto-start service (if installed)
openhoof service uninstall

# Stop the daemon
openhoof stop

# Uninstall the package
npm uninstall -g openhoof

# Optionally remove config and data
rm -rf ~/.openhoof

Troubleshooting

Permission errors on npm install

If you get EACCES errors, fix your npm permissions instead of using sudo:

# Create a directory for global packages
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'

# Add to your shell profile (~/.zshrc or ~/.bashrc)
export PATH=~/.npm-global/bin:$PATH

Port already in use

If port 3777 is taken, specify a different port in your config or via the CLI:

openhoof start --port 3778

Daemon won't start

Run in foreground mode to see the error output:

openhoof start --foreground
